üì° API Endpoints

CVE Operations

GET /api/v1/cves/ - List CVEs with filtering and pagination
GET /api/v1/cves/{cve_id} - Get specific CVE by ID
GET /api/v1/cves/year/{year} - Get CVEs by publication year
GET /api/v1/cves/score/{min_score}/{max_score} - Filter by CVSS score range
GET /api/v1/cves/recent/{days} - Get recently modified CVEs
GET /api/v1/cves/search/ - Full-text search CVEs
GET /api/v1/cves/statistics/ - Get CVE statistics

Synchronization Operations

POST /api/v1/sync/ - Trigger manual synchronization
GET /api/v1/sync/status - Get synchronization status
GET /api/v1/sync/history - Get synchronization history
POST /api/v1/sync/cancel - Cancel running synchronization

System Operations

GET /health - Application health check
GET /info - Application information

üîß Configuration

Configure the application using environment variables or a .env file:

  • SUPABASE_URL: Your Supabase project URL
  • SUPABASE_KEY: Your Supabase anon key
  • SECRET_KEY: Application secret key
  • NVD_API_KEY: Optional NVD API key for higher rate limits

üìä Example Usage

Try these example API calls:

Get recent high-severity CVEs:
GET /api/v1/cves/?min_score=7.0&modified_since=2024-01-01
Search for specific vulnerabilities:
GET /api/v1/cves/search/?q=SQL injection
Get CVE statistics:
GET /api/v1/cves/statistics/
Trigger data synchronization:
POST /api/v1/sync/ {"sync_type": "incremental"}
